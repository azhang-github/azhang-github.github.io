Final Project: Impact of Public Policy on Covid Cases¶

By Ava Zhang¶

Introduction¶

Since emerging in late 2019, COVID-19 has swept across the world, leaving many to wonder how countries' governments should best respond to the crisis. Different countries had different responses. While many countries restricted traveling, closed schools, and instated mandatory working from home, Sweden is notable in its lack of nationwide lockdown and school closings. Citizens were encouraged to stay home when sick, but such recommendations were not enforced and relied soley on voluntary action. A more in-depth description of Sweden's preventative measures can be found here: https://www.nature.com/articles/s41599-022-01097-5.

This project seeks to examine how Sweden's direction for public policy impacted its COVID cases compared to those of other countries who had more restrictive public policies. These countries to compare against must be similar geographically and hava similar GDP per capita if order for this comparison to be meaningful. I decided on comparing Sweden to fellow Nordic countries Finland and Norway, both of whom have more restrictive COVID policies (Finland: https://yle.fi/news/3-11260062, Norway: https://www.nrk.no/norge/alle-utdanningsinstitusjoner-stenges-_-flere-arrangementer-og-virksomheter-far-forbud-1.14940952) and a relatively similar GDP per capita (https://data.worldbank.org/indicator/NY.GDP.PCAP.CD).

Data Collection¶

I wanted to do a project on Sweden's COVID-19 statistics, since Sweden is a country that has had an open-border policy throughout the entirety of the COVID-19 pandemic. The class provided site that lists many different public datasets called https://github.com/awesomedata/awesome-public-datasets. The site is a great resource for any research project, hosting datasets for a variety of topics. On it, I found a dataset on COVID-19 cases in many different countries, which can be reached by going through https://dj2taa9i652rf.cloudfront.net/ > rearc-covid-19-world-cases-deaths-testing/ > json/. The dataset was contained as a json file, which I downloaded as "part-00000-8df47197-4c9a-4f0d-bf4a-d6d43ea1a317-c000.json" It contains columns describing the country, the country code, the date for a particular entry, the total cases in the country at that date, the number of new cases on that date, the total deaths in the country at that date, the number of new deaths that occurred on that date, the total number of cases per million of that country's population, the number of new cases per million of that country's population, the number of total deaths per million population, the number of new deaths per million of population, and testing data.

One of the best ways of storing tabular information like this is in Python DataFrames. DataFrames allow you to easily store, retrieve, and apply transformations on tabular data. For documentation on DataFrames, see https://pandas.pydata.org/docs/reference/api/pandas.DataFrame.html.

Since the original dataset was incredibly large and I only wanted to examine certain countries, I decided against having one DataFrame to contain all rows in the dataset. I'd have to continuously query the entire huge DataFrame for the information on a specific country, which is very inefficient. I instead decided to put the dataset information into a dictionary of DataFrames. The dictionary would have a specific country as the key, and the key's value would be all the rows in the original dataset belonging to that country (minus the columns indicating the country because that is redundant information). To do this, I imported Python's json and pandas packages, the former to read in the json file as a json object and the latter so I can use DataFrames. A more in depth webpage on json and reading in json files is https://docs.python.org/3/library/json.html.

In [1]:
import json
import pandas as pd

I first wanted to read in the json file using the json.load() method. However, there turned out to be a problem in the downloaded json file. While each individual line in the json format, the file itself was formatted incorrectly. Jupyter notebook (the computing platform I was doing this project on) wouldn't load an incorrectly-formatted json file, so I had to convert the json file to a text file (now called "part-00000-8df47197-4c9a-4f0d-bf4a-d6d43ea1a317-c000.txt") and read it line by line using Python's readlines method.

In [2]:
with open('part-00000-8df47197-4c9a-4f0d-bf4a-d6d43ea1a317-c000.txt') as file:
    lines = file.readlines()

For each line the file, we can then read in the row using the json.loads method, converting the row into a json object we can parse for column values. We find the row's location value to figure out which key in the dictionary we should add the row information to. If there doesn't already exist a DataFrame in the dictionary for that specific location, we create a new dataframe out of the column values to save as the location's value. Else, we create a list of the data values that will represent the new row to add to the existing DataFrame for that location.

However, in the dataset, there are json lines that use an entirely different format then the one we assume when extracting the information about total cases and new cases. These are the rows about the aforementioned testing data in the dataset, and when we encounter these lines in the file and try to parse out columns that don't exist in those roles, we raise an error. Since I didn't want to include those rows as information in the dictionary anyway, I created a try-except block to catch these errors and just ignore those rows, i.e. rows that don't match our expected formatting aren't added into the dictionary of DataFrames.

The last formatting errors could occur is for rows of countries that are yet to have any deaths (but still have cases), the columns of total deaths, new deaths, total deaths per million, new deaths per million are excluded from the json line, and thus when my code attempts to parse for those columns it throws an error because it doesn't exist. To account for this error, I use the DataFrame method get to retrieve the value of the column if it is provided and return the default value of 0 deaths if it is not.

This next section of code creates the dictionary of DataFrames as described above.

In [3]:
total_dict = {}

for line in lines:
    data = json.loads(line)
    loc = data["location"]
    
    # If location is not in the dictionary yet, first create a DataFrame with the proper columns and without entries
    if total_dict.get(loc) is None:
        total_dict[loc] = pd.DataFrame(columns = ['date', 'total_cases', 'new_cases', 'total_deaths', 'new_deaths', 'total_cases_per_million', 'new_cases_per_million', \
                                                  'total_deaths_per_million', 'new_deaths_per_million'])

    # If the row doesn't have 'total_cases' columns, then the documentation is about test data, which we don't care about for the purpose of this project
    try:
        # The specific .get(parameter, 0) calls are for the 'death' columns.  In the original dataset, 0 values weren't included as columns at all, so we must include a default value manually
        values = [data['date'], data['total_cases'], data['new_cases'], data.get('total_deaths', 0), data.get('new_deaths', 0), data['total_cases_per_million'], \
                  data['new_cases_per_million'], data.get('total_deaths_per_million', 0), data.get('new_deaths_per_million', 0)]
        total_dict[loc].loc[len(total_dict[loc].index)] = values
    except:
        pass

# This print statement helps me tell when this incredibly time-consuming operation has finished.
print("Finished operation")

Finished operation

Due to the long time it takes for the dataset to be read into the dictionary of DataFrames, I decided to create a copy of the dictionary when doing the rest of the code so if I make a change to the dictionary I want to undo, I can just copy another dictionary from total_dict instead of having to do the entire process of reading the dataset in again. It's good practice to ensure you aren't wasting time waiting for an action you've already performed to re-run if it's not absolutely necessary.

Exploratory Data Analysis¶

Total Cases per Million¶

For the first part of my exploratory data analysis, I want to create a linear regression line for the "total cases per million" graphs. In order to put the dates into a linear regression fit as an independent variable, I had to convert them into a number representation. To do so, I decided to import MatPlotLib's dates package for its date2num method that converts datetime objects into Matplotlib dates. Using that method, I created a new column in each of the country's DataFrames to hold that a row's date's numerical representation.

In [ ]:
import matplotlib.dates
data_copy['Sweden']['date_num'] = matplotlib.dates.date2num(data_copy['Sweden']['date'])
data_copy['Finland']['date_num'] = matplotlib.dates.date2num(data_copy['Finland']['date'])
data_copy['Norway']['date_num'] = matplotlib.dates.date2num(data_copy['Norway']['date'])

To create a linear regression line, I used Sklearn's linear model's LinearRegression function that takes in a set of independent and dependent variables and creates a line of best fit that minimizes the residual sum of squares between the provided data and the predicted line. More information on this function can be found here: https://scikit-learn.org/stable/modules/generated/sklearn.linear_model.LinearRegression.html

In [ ]:
import numpy as np
from sklearn.linear_model import LinearRegression

For each country, we create a Numpy array of x values (the numerical date representation) and another for y values (the total cases per million for that day). The x array must be reshaped before both are put into the Linear Regression's fit function, which will create a Linear Regression object that can predict total cases per million given a date (in numerical representation). We then create a column in each country's DataFrame that holds the expected total cases per million for each date so we can plot the linear regression later.

In [ ]:

sweden_x = np.array(data_copy['Sweden']['date_num'].tolist())

Hypothesis Testing¶

Kolmogorov-Smirnov Test¶

We see that the Sweden distribution tends to have more Covid cases (total or new) through the graphs, but we want to prove this using hypothesis testing. More specifically, I want to prove that Sweden's distribution of total cases per milllion is generally larger than those of Norway and Finland. One way of doing this is using the two-sample Kolmogorov-Smirnov test for goodness of fit. It provides a p-value that represents the likelihood that two samples from the same population would have as extreme (or more extreme) differences than the two samples provided. We will be using a one-tailed variant testing for Sweden having a larger distribution than both Norway and Sweden. The null hypothesis is that two distributions provided are identical, the alternate hypothesis are Finland's or Norway's distributions are from a population with lesser total cases per million than Sweden's. The python package SciPy has an implementation of this test called ks_2smp that I will be using to test the countries' new_cases_per_million distributions. For more information on Kolmogorov-Smirnov or SciPy's implementation of the test, see https://docs.scipy.org/doc/scipy/reference/generated/scipy.stats.ks_2samp.html.

In [ ]:
from scipy.stats import ks_2samp

# alternate = "less" means that we're testing for Finland or Norway to have a smaller distribution than Sweden
print("Sweden vs Finland:", ks_2samp(data_copy['Sweden']['total_cases_per_million'], data_copy['Finland']['total_cases_per_million'], alternative='less', mode='asymp'))
print("Sweden vs Norway:", ks_2samp(data_copy['Sweden']['total_cases_per_million'], data_copy['Norway']['total_cases_per_million'], alternative='less', mode='asymp'))

All of the p values were smaller than the alpha of a = 0.05, indicating that we can reject the null hypothesis of the distributions being from the same population and accept the alternative hypothesis that Sweden has a generally larger distribution of total cases per million than Finland or Norway.

Polynomial Regression Fitting¶

In addition to the evidence from looking at the Kolmogorov-Smirnov Test, I will also be fitting the "total cases per million" distributions of the Nordic countries to 2nd degree polynomial regressions, since it functions as a continuous distribution. I believe that, with restrictive policies like Finland's or Norway's, countries will have a slow, quadratic regression. However, countries that have a more open policy (like Sweden) will be more unpredictable, since one person with COVID could spread it to considerably more people since they aren't forced to quarantine, causing more dates of steep increases and thus causing the distribution to not match its polynomial regressions as well as Finland's and Norway's would. We can do a least square polynomial fit using NumPy's polyfit() method, which takes in the independent and dependent variables to fit on, as well as the number of dimensions we will be fitting on. The result returned is the coefficients of the model (more information on it can be found at https://numpy.org/doc/stable/reference/generated/numpy.polyfit.html).

Finland¶

First, to fit the 2-dimensional polynomial regression we take the values that will act as the dependent variable y (total cases per million) and x (the numerical representations of the date) and put it in the function. We can then put the resulting polynomial coefficients into NumPy's poly1d() function, which produces a polynomial class that can predict a country's total cases per million based on a provided date. (More information on poly1d can be found at https://numpy.org/doc/stable/reference/generated/numpy.poly1d.html).

However, the first thing we must do is split the x and y values into training and testing sets; we don't want to create the regression on all of our data in case we overfit and create a regression that does not accurately describe the phenomenon but just that particular data set. We will split our data values using Sklearn's model selection's train_test_split function, which splits arrays into random train and test subsets. We will use 80% of the data for training and 20% of the data for testing. We specify shuffle as true since currently the data is sorted in order of date, and we want a sampling of random dates. You can find more information on the train_test_split at https://scikit-learn.org/stable/modules/generated/sklearn.model_selection.train_test_split.html.

from sklearn.model_selection import train_test_split
